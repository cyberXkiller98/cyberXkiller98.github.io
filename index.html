    Это усложняет клиентское приложение и может вызвать необоснованный заказ товаров несколькими приложениями.

a.

b.

RDA

c.

d.

Текст вопроса

Выберите свойства, присущие DBS-модели клиент-серверной архитектуры.

Выберите один или несколько ответов:
a.

 Для отслеживания текущего состояния информационного хранилища на сервере используется механизм триггеров и и механизм ограничений на пользовательские типы данных, называемый механизмом поддержки доменной структуры.

b.

Клиентское приложение обращается к серверу с командой запуска хранимой процедуры, а сервер выполняет эту процедуру и регистрирует все изменения в БД, которые в ней предусмотрены. Сервер возвращает клиенту данные, релевантные его запросу либо для вывода на экран, либо для выполнения части бизнес-логики, которая расположена на клиенте.

c.

Запрос клиента формулируется в командах ЯМД. СУБД на клиенте переводит этот запрос в последовательность файловых команд. Каждая файловая команда вызывает перекачку блока информации на клиента, далее на клиенте СУБД анализирует полученную информацию, и если в полученном блоке не содержится ответ на запрос, то принимается решение о перекачке следующего блока информации.

d.

Процесс, выполняемый на компьютере-клиенте, ограничивается функциями представления, а прикладные функции реализованы в хранимых процедурах (Stored Procedure - SP). SP находятся непосредственно в базе данных и выполняются на компьютере-сервере (где функционирует ядро СУБД).

e.

Вводится дополнительный промежуточный уровень между клиентом и сервером - серверы приложений. Они спроектированы для исполнения общих незагружаемых функций для клиентов. Они поддерживают функции взаимодействия клиентов, сетевую доменную операционную среду, каталоги с данными, хранят и исполняют общие правила бизнес-логики.

Ваш ответ верный.
Правильные ответы: Процесс, выполняемый на компьютере-клиенте, ограничивается функциями представления, а прикладные функции реализованы в хранимых процедурах (Stored Procedure - SP). SP находятся непосредственно в базе данных и выполняются на компьютере-сервере (где функционирует ядро СУБД).,  Для отслеживания текущего состояния информационного хранилища на сервере используется механизм триггеров и и механизм ограничений на пользовательские типы данных, называемый механизмом поддержки доменной структуры., Клиентское приложение обращается к серверу с командой запуска хранимой процедуры, а сервер выполняет эту процедуру и регистрирует все изменения в БД, которые в ней предусмотрены. Сервер возвращает клиенту данные, релевантные его запросу либо для вывода на экран, либо для выполнения части бизнес-логики, которая расположена на клиенте.
